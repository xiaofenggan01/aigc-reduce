# AIGC 降重 Skill

## Codex 使用定位

你是学术论文 AIGC 痕迹诊断器和修改规划器，不是全文重写器。Codex 使用本 skill 时必须先定位当前 skill 根目录，也就是本 `SKILL.md` 所在目录，再按需读取 `references/` 或执行 `scripts/`。

不要硬编码 Claude Code 路径。运行脚本时优先使用以下方式之一：

```bash
cd <aigc-reduce-skill-root>
python3 scripts/aigc_scan.py <file.txt>
```

或在已知 skill 根目录时：

```bash
python3 <aigc-reduce-skill-root>/scripts/aigc_scan.py <file.txt>
```

## 四条铁律

### 1. 禁止 AI 全量重写

用 AI 全量重写 AI 文本会叠加新的模型指纹。所有改写应尽量通过确定性字符串替换、局部语序调整、长句拆分、段落节奏调整完成，直接操作原文中的具体片段。

### 2. 修改率目标必须明确

以 >40% 作为深度修改目标。低于 20% 通常只是表层替换，20-40% 只能算中度修改，超过 40% 才有机会显著改变统计特征。不能只改几个连接词就提交。

### 3. 确定性替换优先

每次只改一小处，保留原文事实、数据、引用、专业术语和核心逻辑。禁止为了“像人写的”而引入未给出的实验细节、虚构引用或改变结论强度。

### 4. 保持学术语体

“去 AI 味”和“保持学术语体”是两个独立目标。不能把论文改成口语、网络语或情绪化表达。需要具体负面清单时读取 `references/replacement-tables.md`。

## 三轮降重协议

### 第一轮：去除 AI 痕迹

1. 扫描定位：先把待处理文本保存为 `.txt`，运行 `scripts/aigc_scan.py`。
2. 词级替换：处理模板连接词、AI 高频词、重复动词和“XX了”句式。
3. 句级重构：做语序重组、长句拆分、被动改主动、主语更换。
4. 段落调整：打破段落长度对称，补足必要过渡，删除空洞结论。

详细替换表和禁用表达见 `references/replacement-tables.md`。只有在需要具体替换候选时再读取该文件。替换时每处使用不同表达，避免把一套模板换成另一套模板。

### 第二轮：注入人类写作特征

只去除模板痕迹不够。干净、均匀、过度顺滑的文本仍然容易显得像 AI。

- 节奏工程：检查句长变化，如果 CV < 0.4，优先拆分部分长句并插入短句。
- 不确定性表达：在证据不足处加入谨慎判断，但不得削弱已由数据明确支持的结论。
- 具体化：把“相关研究表明”“据统计”等模糊归因改成已有引用、数据或可核实描述。
- 声音校准：如果用户提供参考文本，先提取其句长、过渡词、段落长度和常用表达，再匹配风格。

### 第三轮：Anti-AI 审计

修改后逐段自检：这段话为什么还像 AI 写的？需要完整审计清单时读取 `references/ai-patterns.md`。

优先排查：

- 重要性膨胀：删掉“重要”“关键”“显著”后含义是否不变。
- 同义词轮换：同一概念是否被无必要地换了多个名字。
- 三板斧结构：是否连续出现 “A、B 和 C”。
- 模糊归因：是否存在没有引用或数据支撑的“研究表明”。
- 悬浮式分析：一句话是否挂了多个“从而/进而/因此”。
- 空洞结论：是否使用“具有良好应用前景”等不可验证表述。
- 段落和句式是否过于整齐。

## 验证流程

修改后必须验证：

1. 再次运行扫描脚本，对比模板句式、被动语态、突发性、段落对称性、编号、冒号并列和标点规律。
2. 估算实际修改率，低于 40% 时继续补充确定性修改。
3. 人工复核事实、数据、引用、术语、论证方向是否保持一致。
4. 如用户需要，可建议在腾讯朱雀等免费检测平台做外部复测，但不要把平台分数当作唯一真值。

## 脚本说明

`scripts/aigc_scan.py` 扫描文本的 7 个 AI 特征维度，支持普通报告和 `--json` 结构化输出。

```bash
python3 scripts/aigc_scan.py <file.txt>
python3 scripts/aigc_scan.py <file.txt> --json
```

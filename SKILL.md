# AIGC 降重 Skill

## 核心原则

### 铁律：禁止用 AI 全量重写

> **用 AI 重写 AI 文本 = 叠加 AI 指纹，会使检测率更高。**
>
> 实测验证：一篇论文经过 Claude 全量重写后，PaperPure 检测 AI 率从原始值升至 100%。
> 原因：AI 模型在生成时总是选择概率最高的 token，重写后的文本困惑度更低、突发性更小、
> 句式更均匀——恰好是检测器判定"更像 AI"的特征。这不是消除指纹，而是在原始 AI 指纹上
> 叠加了新的 Claude/GPT 指纹。

### 正确策略：扫描定位 + 确定性替换 + 最小化干预

Claude 的角色是**诊断器**（扫描、定位、分析），而非**重写器**（全量生成新文本）。

改写应通过**确定性字符串替换**（直接操作原文的特定片段）完成，而非让 LLM 重新生成整段文本。
这避免了 token 采样过程中的 AI 指纹叠加。

```
❌ 错误：把原文整段丢给 Claude，让它重写 → AI 指纹叠加 → 检测率更高
✅ 正确：扫描定位高危片段 → 用字符串替换规则定点修改 → 保留原文的大部分 token
```

## 降重方法（按效果由浅到深）

### 第一层：确定性替换（优先使用）

不经过 LLM 生成，直接对原文做字符串级操作。这种方式不引入新的 AI 指纹。

1. **模板连接词替换表** — 查找并替换固定模式：
   - "此外" → "另一方面"/"另一组数据则显示"/直接删除
   - "与此同时" → "另一方面"/"同时"/"结合前文来看"
   - "综上所述" → 删除，或改为具体判断
   - "需要指出的是" → "值得注意的是"/"值得说明的是"/直接删除
   - "研究发现"/"研究表明" → "实验表明"/"数据呈现"/"从结果来看"
   - **每处替换必须不同**，不能把"此外"全文统一替换成"另一方面"

2. **"XX了"高危句式替换表**：
   - "揭示了" → "指向了"/"表明"/"体现了"
   - "探讨了" → "考察了"/"分析了"/"梳理了"
   - "验证了" → "确认了"/"证明了"/"实测结果表明"
   - "构建了" → "建立了"/"搭建了"/"设计出"
   - "展示了" → "呈现了"/"反映了"/"体现了"
   - "证实了" → "确认了"/"验证了"/"实测支持了"
   - **每处替换必须不同**

3. **语序重组（最有效的确定性操作）** — 不改词，只调换句子成分顺序：
   - "研究某某领域的某某事物" → "对某某领域的某某事物进行研究"
   - "选择某个方式针对某个事物研究，从而得出某个结论" → "为了得出某个结论，选择某个事物作为研究对象，通过某个方式来研究"
   - 倒装、前置、后置、拆分都可以

4. **排比/并列结构拆解** — 用确定性规则拆解：
   - `（1）…（2）…（3）…` → 改为自然叙述，用不同的过渡词连接
   - `A：…；B：…；C：…` → 拆成独立叙述段落
   - "首先…其次…最后…" → 删除序号标记，改为自然衔接

5. **标点微调** — 混用破折号、冒号、引号，打破 AI 的全角标点工整排列

### 第二层：结构微调（低干预）

需要少量 LLM 辅助，但只针对特定片段做最小化修改。

6. **拆长句为短句** — 将 50 字以上的单句在合适位置断开，加句号
7. **被动改主动** — "被测定为" → "测得"
8. **打破对称段长** — 连续 3 段以上长度相近时，将其中一段适当扩写或精简
9. **加入段落间过渡** — 每段结尾加一句引导，下段开头承接上文

### 第三层：语义注入（高干预，仅用于核心段落）

需要 LLM 生成新内容时，**只注入少量片段**，不要重写整段。

10. **嵌入操作观察** — 在数据和结论之间插入实验细节（1-2 句即可）：
    - "过程中发现粉料容易结团，需分次加水"
    - "反应时间浮动 ±15 min"
    - "因高黏度物料易过载，搅拌电流上升约 20-30%"
11. **加入作者判断** — 在客观陈述后插入 1 句主观判断
12. **具体化模糊表述** — 删除"据统计""相关研究表明"，或给出具体数值

### 第四层：专项针对

13. **文献综述** — 避免"A 认为…B 指出…C 提出…"的并列，改为有主次、有批判的叙评结合
14. **结论与展望** — 避免对称列举，合并叙述，加入对实际约束的讨论
15. **实验方法** — 加入实操视角表述

## 改写操作规范

### 必须遵守

1. **优先使用确定性替换**（第一层），避免 LLM 全量生成
2. **每次操作只改一小处**，保留原文的大部分 token 不变
3. **每次处理不超过 1000 字**，段落间保持差异化
4. **相邻段落的句式结构、开头方式、长度必须不同**
5. **所有数据、引用、专业术语、逻辑关系必须与原文一致**

### 禁止操作

- **禁止用 AI 全量重写整段文本** — 这是最重要的规则
- **禁止引入新的模板句式** — 不能把一套模板换成另一套模板
- **禁止过度口语化** — 学术论文有底线
- **禁止破坏专业术语** — 核心概念的准确表述不能扭曲

## 高危词/句式清单

改写前，先搜索论文中以下高危模式：

### 句式类（最危险）

- "揭示了""探讨了""验证了""阐释了""构建了""展示了""归纳了""对比了""诠释了""推导了""整合了""证实了""凸显了""重构了""模拟了""量化了""追溯了" — "XX了"后跟长句的模式是 AI 强特征
- 叠词形容词串 — 删成简约表述
- "和""或""与""同""、" 串联的超长复合句 — 拆短

### 连接词类

- "首先…其次…最后…"
- "此外""与此同时""综合来看""综上所述""需要指出的是"
- "研究发现""研究表明""分析认为"
- 以上全部删除或替换为不重复的过渡方式

### "了"字清理

- 搜索"了"，删除不必要的口语化"了"
- 正常使用的"了"（"升高了 20%"）保留

## 工作流程

### 步骤 1：扫描定位

调用扫描脚本获取客观数据：

```bash
python3 ~/.claude/skills/aigc-reduce/scripts/aigc_scan.py <file.txt>
```

扫描输出 7 个维度：模板句式密度、被动语态比例、句子长度变化（突发性 CV）、段落对称性、嵌套编号、冒号并列结构、标点规律。

### 步骤 2：高危模式排查

在文本中搜索高危模式（见"高危词/句式清单"），输出具体位置和数量。

### 步骤 3：确定性替换（主要操作）

根据步骤 1 和 2 的结果，用确定性字符串替换处理：
- 模板连接词 → 用替换表逐项替换，每处用不同的替代词
- "XX了"句式 → 用替换表逐项替换
- 排比/并列结构 → 用规则拆解
- 语序重组 → 调换句子成分顺序

**关键**：这些操作直接修改原文件的特定字符串，不经过 LLM 的 token 采样。

### 步骤 4：结构微调（按需）

如果步骤 3 之后仍有问题段落：
- 拆长句、被动改主动
- 打破对称段长
- 加入段落间过渡

### 步骤 5：语义注入（仅用于检测报告标注的高风险段落）

对仍被判为 AI 的高风险段落，注入少量人类写作特征：
- 1-2 句操作观察
- 1 句作者判断
- 删除或具体化模糊表述

### 步骤 6：验证

改写后再次扫描，对比各维度指标变化。建议同时在免费检测平台验证：
- 腾讯朱雀：https://matrix.tencent.com/ai-detect/ai_gen_txt
- 对比改写前后在目标检测器上的结果

## 检测平台注意事项

- **PaperPure**：检测灵敏度极高，用 AI 全量重写会被判 100%。只能用确定性替换策略
- **知网 3.0**：准确率 98.6%，重点抓模板句式和格式规范性
- **维普**：英文论文检测可能有虚高问题；2026.5 后检测异常严格
- **PaperYY**：社区反映存在故意虚标 AI 率以推销付费降重的情况
- **腾讯朱雀**：免费，适合修改后快速验证
- **不同平台结果差异大**：同一篇论文知网 35%、维普 18%、万方 12%，属正常现象
- **写得越好越容易被判 AI**：高质量流畅文本假阳性率 6.5%，有口语化瑕疵的文本仅 1.8%

## 脚本说明

### `scripts/aigc_scan.py`

自动扫描文本的 7 个 AI 特征维度，输出风险报告。支持 `--json` 输出结构化数据。

扫描维度对应检测器原理：
| 维度 | 对应检测器 |
|------|-----------|
| 模板句式密度 | 知网语义指纹（模板化句式） |
| 突发性 CV | GPTZero 核心算法（困惑度+突发性） |
| 段落对称性 | 知网格式规范性校验 |
| 嵌套编号 | 统计检测（结构模板化） |
| 冒号并列 | 知网逻辑断层检测 |
| 被动语态 | Turnitin 句式特征分析 |
| 标点规律 | 万方 BERT 语义分析 |

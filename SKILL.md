# AIGC 降重 Skill

## 核心原则

### 铁律 1：禁止用 AI 全量重写

> **用 AI 重写 AI 文本 = 叠加 AI 指纹，会使检测率更高。**
>
> 实测验证：一篇论文经过 Claude 全量重写后，PaperPure 检测 AI 率从原始值升至 100%。
> 原因：AI 模型在生成时总是选择概率最高的 token，重写后的文本困惑度更低、突发性更小、
> 句式更均匀——恰好是检测器判定"更像 AI"的特征。

### 铁律 2：修改率必须超过 40%

> **研究报告核心数据：修改比例与检测规避率直接相关**
>
> | 修改程度 | 假阴性率（检测器漏检率） | 说明 |
> |---------|----------------------|------|
> | 轻度修改（<20%） | 15-25% | 几乎无效 |
> | 中度修改（20-40%） | 35-50% | 有一定效果 |
> | **深度修改（>40%）** | **60-80%** | **目标阈值** |
>
> 第一轮确定性替换只改了约 3% 的文本，远低于阈值。必须进行大规模、系统性的替换。

### 铁律 3：确定性替换，不是 LLM 重写

> Claude 的角色是**诊断器 + 规划器**（扫描、定位、规划替换方案），而非**重写器**。
>
> 所有改写操作必须是**确定性字符串替换**（直接操作原文的特定片段），
> 而非让 LLM 重新生成整段文本。这避免了 token 采样过程中的 AI 指纹叠加。

## 修改率达标策略

### 为什么之前失败？

第一轮确定性替换只做了 15 处字符串替换（约 3% 修改率），远低于 >40% 阈值。
原因：
1. 只做了词级替换（模板连接词、"XX了"句式），没有触及句子结构
2. 替换表太小，覆盖面不够
3. 没有做句子级重构（语序重组、长句拆分）

### 如何达到 40%+ 修改率？

需要**三管齐下**，对文本的 40% 以上字符进行确定性替换：

1. **词级替换**（贡献约 10-15%）：模板词、连接词、动词、形容词的系统性替换
2. **句级重构**（贡献约 15-20%）：语序重组、长句拆分、被动改主动、主语更换
3. **段落级调整**（贡献约 10-15%）：段间过渡、段落扩写/精简、结构重组

三者叠加可达到 40-50% 的修改率。

## 降重方法（按操作粒度）

### 一、词级大规模替换（贡献 ~10-15% 修改率）

**核心思路**：不是替换几个模板词，而是对文本中的大量词语进行系统性替换。

#### 1. 模板连接词替换表

查找并替换，**每处替换必须不同**：

- "此外" → "另一方面" / "另一组数据则显示" / 直接删除 / "从另一角度看"
- "与此同时" → "另一方面" / "同时" / "结合前文来看" / "与这一趋势类似"
- "综上所述" → 删除 / 改为具体判断 / "总体来看" / "从以上分析可知"
- "需要指出的是" → "值得注意的是" / "值得说明的是" / 直接删除
- "研究发现" / "研究表明" → "实验表明" / "数据呈现" / "从结果来看" / "测试结果显示"
- "分析认为" → "推测" / "可能的原因是" / "作者判断" / "这一现象说明" / "据此推断"
- "具体而言" → "从数据来看" / "进一步分析发现" / "拆开来看"
- "值得注意的是" → "有意思的是" / "特别之处在于" / "一个细节是"

#### 2. "XX了"高危句式替换表

- "揭示了" → "指向了" / "表明" / "体现了"
- "探讨了" → "考察了" / "分析了" / "梳理了"
- "验证了" → "确认了" / "证明了" / "实测结果表明"
- "构建了" → "建立了" / "搭建了" / "设计出"
- "展示了" → "呈现了" / "反映了" / "体现了"
- "证实了" → "确认了" / "验证了" / "实测支持了"
- "导致了" → "造成了" / "引发了" / "使得"
- "表明了" → "显示出" / "意味着" / "提示"
- **每处替换必须不同**

#### 3. 动词替换扩展表

- "显著" → "明显" / "大幅" / "可观地" / "突出地"
- "增加" → "上升" / "提高" / "攀升" / "增长"
- "降低" → "下降" / "减小" / "削弱" / "回落"
- "提升" → "增强" / "改善" / "优化" / "促进"
- "影响" → "改变" / "调控" / "左右" / "作用"
- "反映" → "体现" / "呈现" / "揭示" / "展现"
- "促进" → "推动" / "驱动" / "有利于" / "助力"
- "抑制" → "限制" / "阻碍" / "约束" / "削弱"
- "归因于" → "来源于" / "与…有关" / "主要在于" / "原因之一是"
- "呈现" → "表现出" / "显示出" / "具有"
- "认为" → "判断" / "推测" / "倾向于" / "倾向于认为"

#### 4. 形容词/副词替换表

- "显著" → "明显" / "大幅" / "可观地"
- "明显" → "显著" / "清晰可见" / "不难看出"
- "进一步" → "更深入地" / "在更高层面上" / 删除
- "有效" → "切实" / "确实" / "能够"
- "整体" → "从宏观角度" / "全局来看" / "汇总来看"
- "相对" → "比较" / "在…范围内" / "一定程度上"
- "基本" → "总体上" / "大致" / "多数情况下"

#### 5. 句式模式替换表

- "归因于两方面，首先…其次…" → "主要有两方面原因：一是…二是…"
- "这一现象表明…" → "由此可以推断…" / "这说明…"
- "上述结果一致表明" → "综合这几组数据" / "把以上结论放在一起看"
- "与…相比" → "对比…" / "相较于…" / "不同于…"
- "…的影响达到了显著水平" → "…的影响在统计上显著" / "…带来的变化具有统计意义"
- "这说明…" → "其含义是…" / "背后的逻辑是…" / "意味着…"

### 二、句子级重构（贡献 ~15-20% 修改率）

**核心思路**：不换词，但打乱句子结构。这是突破 40% 修改率的关键。

#### 6. 语序重组（最有效的确定性操作）

- "研究某某领域的某某事物" → "对某某领域的某某事物进行研究"
- "选择某个方式针对某个事物研究，从而得出某个结论" → "为了得出某个结论，选择某个事物作为研究对象，通过某个方式来研究"
- 倒装、前置、后置、拆分都可以
- "A 导致 B" → "B 的出现与 A 密切相关" / "A 是 B 产生的重要原因"
- "X 具有 Y 特性" → "Y 是 X 的一个重要特征" / "X 在 Y 方面表现出色"

#### 7. 拆长句为短句

- 将 50 字以上的单句在合适位置断开，加句号
- 复合句拆成 2-3 个独立句
- "由于A，且B，因此C" → "A 和 B 同时存在。在这种条件下，C 自然发生。"

#### 8. 被动改主动

- "被测定为" → "测得"
- "被观察到" → "观察到"
- "被认为是" → "通常看作" / "学界普遍认同"
- "被发现" → "实验数据呈现" / "可以检测到"
- "已被证实" → "已有研究支持" / "实验反复确认"

#### 9. 主语更换

- "实验结果显示" → "从数据中可以看到" / "各组数据对比后不难发现"
- "研究表明" → "已有文献记录" / "多篇论文的数据指向"
- "本研究发现" → "测试过程中一个有趣的观察是" / "从实验结果中可以提取出"
- "结果显示" → "对比各组数据后" / "把数据放在一起审视"

#### 10. 排比/并列结构拆解

- `（1）…（2）…（3）…` → 改为自然叙述，用不同的过渡词连接
- `A：…；B：…；C：…` → 拆成独立叙述段落
- "首先…其次…最后…" → 删除序号标记，改为自然衔接
- "一方面…另一方面…" → 改为"不仅…也…"或自然叙述

### 三、段落级调整（贡献 ~10-15% 修改率）

#### 11. 打破对称段长

- 连续 3 段以上长度相近时，将其中一段适当扩写或精简
- 扩写手段：插入操作观察、补充数据细节
- 精简手段：合并冗余表述、删除重复结论

#### 12. 加入段落间过渡

- 每段结尾加一句引导，下段开头承接上文
- "这一发现也引出了下一个问题："
- "与前述力学性能相对应的，是"
- "从结构表征转向功能评价，"

#### 13. 语义注入（仅用于高风险段落）

在数据和结论之间插入实验细节（1-2 句）：
- 操作观察："过程中发现粉料容易结团，需分次加水"
- 参数波动："反应时间浮动 ±15 min"
- 设备细节："因高黏度物料易过载，搅拌电流上升约 20-30%"
- 异常说明："第 3 批数据偏高，疑因环境温度波动"

加入作者判断（1 句主观判断）：
- "这一现象值得注意"
- "实际应用中需权衡"
- "这提示我们在配方设计中应"

具体化模糊表述：
- 删除"据统计""相关研究表明"
- 给出具体数值或来源

#### 14. 专项针对

- **文献综述**：避免"A 认为…B 指出…C 提出…"的并列，改为有主次、有批判的叙评结合
- **结论与展望**：避免对称列举，合并叙述，加入对实际约束的讨论
- **实验方法**：加入实操视角表述
- **数据分析**：加入对数据趋势的主观解读

## 改写操作规范

### 必须遵守

1. **修改率必须 >40%**：对文本中至少 40% 的字符进行确定性替换
2. **优先使用句子级重构**（语序重组、长句拆分），这是修改率的主力来源
3. **每次操作只改一小处**，保留原文的核心 token
4. **相邻段落的句式结构、开头方式、长度必须不同**
5. **所有数据、引用、专业术语、逻辑关系必须与原文一致**

### 禁止操作

- **禁止用 AI 全量重写整段文本** — 这是最重要的规则
- **禁止引入新的模板句式** — 不能把一套模板换成另一套模板
- **禁止过度口语化** — 学术论文有底线
- **禁止破坏专业术语** — 核心概念的准确表述不能扭曲
- **禁止替换率低于 40% 就提交** — 低于阈值几乎无效

## 高危词/句式清单

改写前，先搜索论文中以下高危模式：

### 句式类（最危险）

- "揭示了""探讨了""验证了""阐释了""构建了""展示了""归纳了""对比了""诠释了""推导了""整合了""证实了""凸显了""重构了""模拟了""量化了""追溯了" — "XX了"后跟长句的模式是 AI 强特征
- 叠词形容词串 — 删成简约表述
- "和""或""与""同""、" 串联的超长复合句 — 拆短

### 连接词类

- "首先…其次…最后…"
- "此外""与此同时""综合来看""综上所述""需要指出的是"
- "研究发现""研究表明""分析认为"
- 以上全部删除或替换为不重复的过渡方式

### 动词重复类

- 同一个动词出现 3 次以上 → 必须替换其中至少 2 次
- "分析认为"出现多次 → 每次用不同替代

### "了"字清理

- 搜索"了"，删除不必要的口语化"了"
- 正常使用的"了"（"升高了 20%"）保留

## 深度 AI 痕迹识别（源自 Wikipedia AI Writing Patterns）

> 以下模式提炼自 Wikipedia "Signs of AI writing" 指南（由 WikiProject AI Cleanup 维护），
> 并针对中文学术写作场景做了本地化适配。这些模式超出了传统统计检测的范畴，
> 即使困惑度和突发性正常，以下痕迹仍可能被深度学习检测器（如 PaperPure）捕捉。

### 1. 重要性膨胀（Significance Inflation）

AI 倾向给普通事实加上"重大意义"包装，制造不存在的重要性感。

**高危词：** "至关重要""不可忽视""深远影响""具有重要价值""为…提供了重要参考""对…具有重要意义""标志着""开启了新篇章""推动了…的发展"

**识别方法：** 检查每个结论句——如果删掉"重要""关键""显著"等修饰词后句子含义不变，说明这些修饰词是 AI 添加的膨胀内容。

**处理：** 删除膨胀修饰，或替换为具体数据。例如：
- "对食品工业具有重要意义" → "在植物基仿生食品开发中有应用潜力"
- "为该领域提供了重要的理论依据" → "为 SPI-KGM-KC 凝胶的配方设计提供了参考"

### 2. 同义词轮换（Synonym Cycling）

AI 因 repetition penalty 会过度替换同义词，导致同一概念在一篇论文中出现多个不同称呼。人类作者通常全文统一使用一个术语。

**检测模式：**
- 同一个实验对象被交替称为"凝胶""胶体""水凝胶""复合物"
- 同一个指标被交替称为"硬度""坚实度""抗变形能力""机械强度"
- 同一个操作被交替称为"测定""检测""分析""表征""评估"

**处理：** 核心术语全文统一，只在修辞需要时才做变化。如果原文已经统一，不要人为制造轮换。

### 3. 三板斧强迫症（Rule of Three）

AI 强迫把论点凑成三个一组，人类则自然地使用不同数量的并列项。

**检测模式：**
- "XXX、YYY和ZZZ" 且三项长度相近
- "不仅A，而且B，甚至C" 的三段递进
- 每个段落恰好讨论三个要点
- 连续多段都是"A、B和C"的结尾模式

**处理：** 将部分三件套拆为两件（合并两项）或四件（拆分一项），打破规律。有些地方二件并列比三件更自然。

### 4. 系词回避（Copula Avoidance）

AI 不愿意用简单的"是"，偏用更复杂的系动词。

**检测模式：**
- "作为…的代表" → 就是"是…的代表"
- "体现了…" → 就是"是…"
- "扮演着…的角色" → 就是"是…"
- "可视为" / "可被理解为" / "本质上是"
- "充当了" / "起到了…的作用"

**处理：** 该用"是"的地方就用"是"，学术写作中"是"并不低级。

### 5. 模糊归因（Vague Attributions）

AI 把观点归于模糊的权威而不给出具体引用。

**检测模式：**
- "有研究表明" 但后面没有引用编号
- "学界普遍认为" 无出处
- "众所周知" 无佐证
- "一般认为" 无参考文献
- "相关研究指出" 无具体文献

**处理：** 要么删掉模糊归因（让陈述直接作为作者自己的结论），要么给出具体引用编号或数据来源。

### 6. 公式化挑战段（Formulaic Challenges Section）

AI 在结论/讨论部分使用固定模板承认局限性。

**检测模式：**
- "尽管取得了…成果，但本研究仍存在一定的局限性"
- "未来的研究可以进一步探讨…"
- "本研究存在以下不足：第一…第二…第三…"
- 这些段落的结构和用词在大量论文中几乎一模一样

**处理：** 用自然叙述替代模板。例如：
- "需要指出的是，本实验只在实验室条件下测试了三种 SPI 浓度，工业化生产中的表现还需进一步验证"（具体、自然）
- 而非 "尽管本研究取得了一定成果，但仍存在局限性，未来研究可以进一步扩大样本量并考虑更多实际因素"

### 7. 悬浮式分析（Superficial -ing Analyses）

AI 在陈述事实后附加"从而…""进而…""由此…"等悬挂式短语，制造虚假的分析深度。

**检测模式：**
- "…从而实现了…"（结尾性悬挂）
- "…进而促进了…"（递进性悬挂）
- "…由此可知…"（推论性悬挂）
- 一句话末尾挂了 2 个以上的"从而/进而/由此"

**处理：** 将悬挂短语独立成句，或删掉不提供实质信息的部分。例如：
- "KC浓度升高增强了KGM链间的氢键交联，从而使凝胶由疏松网络向致密连续网络转变" → "KC浓度升高后，KGM链间的氢键交联更密集。凝胶从松散网络转变为紧密连续的结构。"

### 8. 空洞结论（Generic Positive Conclusions）

AI 在段落末尾给出空洞的正面总结。

**检测模式：**
- "具有良好的应用前景"
- "为…提供了理论和实验依据"
- "为…的开发奠定了基础"
- "具有重要的理论价值与现实意义"
- 这些表述可以套用到任何论文的任何段落

**处理：** 替换为具体的判断。例如：
- "具有良好的应用前景" → "该配比下的凝胶硬度接近动物性食品牛肚的实测值（约 3.2 kPa），可作为素毛肚的原型材料"
- 空洞结论 = 检测器最容易捕捉的 AI 特征之一

### 9. 破折号过度使用（Em Dash Overuse）

AI 使用破折号（——）的频率远高于人类作者。

**检测模式：** 一段话中出现 2 个以上"——"
- "这一现象——虽然在预料之中——仍然值得关注"
- "KC浓度的变化——尤其是从C1到C2——导致了…"

**处理：** 大部分破折号可以替换为逗号、句号或括号。一段话中最多保留 1 个破折号。

### 10. "从X到Y"虚假范围（False Ranges）

AI 使用"从X到Y"结构来制造广度感，但X和Y往往不在同一量度上。

**检测模式：**
- "从宏观到微观""从理论到实践""从基础研究到工程应用"
- "涵盖了从A到B的各个方面"

**处理：** 直接列举实际覆盖的范围，或删掉。

## 工作流程

### 步骤 1：扫描定位

调用扫描脚本获取客观数据：

```bash
python3 ~/.claude/skills/aigc-reduce/scripts/aigc_scan.py <file.txt>
```

扫描输出 7 个维度：模板句式密度、被动语态比例、句子长度变化（突发性 CV）、段落对称性、嵌套编号、冒号并列结构、标点规律。

### 步骤 2：评估修改率

统计原文总字符数。规划替换方案时，估算每种操作的修改字符量：

```
词级替换（10-15%）：约 600-900 字符 / 6000 字文本
句级重构（15-20%）：约 900-1200 字符 / 6000 字文本
段落级调整（10-15%）：约 600-900 字符 / 6000 字文本
─────────────────────────────────────────────
合计目标：40-50% 修改率
```

**如果规划替换方案的总修改率不足 40%，必须补充更多替换操作。**

### 步骤 3：执行确定性替换

按照「词级 → 句级 → 段落级」的顺序，逐项执行确定性字符串替换。

**关键**：这些操作直接修改原文件的特定字符串，不经过 LLM 的 token 采样。

对于 docx 文件，通过 python-docx 定位段落并替换文本。

### 步骤 4：验证修改率

改写完成后统计实际修改率：
- 对比原文和修改后的文本
- 计算被修改的字符数 / 总字符数
- 如果 <40%，继续补充替换操作

### 步骤 5：二次自检（Anti-AI Pass）

> 源自 humanizer skill 的核心方法论：改完不是结束，必须再做一轮审查。

执行方式：
1. 审视修改后的文本，回答：**"这段话为什么看起来还像 AI 写的？"**
2. 列出剩余的 AI 痕迹（参考「深度 AI 痕迹识别」中的 10 种模式）
3. 针对列出的痕迹进行第二轮确定性替换
4. 重复直到找不到明显的 AI 痕迹

**自检重点清单：**
- [ ] 是否还有重要性膨胀？（删掉"重要""关键"后含义不变？）
- [ ] 是否有同义词轮换？（同一概念被叫了 3 个以上名字？）
- [ ] 是否有三板斧？（连续出现"A、B和C"三件套？）
- [ ] 是否有空洞结论？（"具有良好的应用前景"？）
- [ ] 是否有公式化挑战段？（"尽管…但仍存在局限性"？）
- [ ] 是否有模糊归因？（"有研究表明"无引用？）
- [ ] 是否有悬浮式分析？（一句话挂了 2+ 个"从而/进而"？）
- [ ] 段落长度是否过于均匀？（突发性太低？）
- [ ] 句式结构是否过于整齐？（缺少长短句交错？）

### 步骤 6：再次扫描对比

改写后再次扫描，对比各维度指标变化。建议同时在免费检测平台验证：
- 腾讯朱雀：https://matrix.tencent.com/ai-detect/ai_gen_txt
- 对比改写前后在目标检测器上的结果

## 检测平台注意事项

- **PaperPure**：检测灵敏度极高，用 AI 全量重写会被判 100%。只能用确定性替换策略，且修改率必须 >40%
- **知网 3.0**：准确率 98.6%，重点抓模板句式和格式规范性
- **维普**：英文论文检测可能有虚高问题；2026.5 后检测异常严格
- **PaperYY**：社区反映存在故意虚标 AI 率以推销付费降重的情况
- **腾讯朱雀**：免费，适合修改后快速验证
- **不同平台结果差异大**：同一篇论文知网 35%、维普 18%、万方 12%，属正常现象
- **写得越好越容易被判 AI**：高质量流畅文本假阳性率 6.5%，有口语化瑕疵的文本仅 1.8%

### `scripts/aigc_scan.py`

| 标点规律 | 万方 BERT 语义分析 |

# AIGC Reduce

降低学术论文 AIGC 查重率的 Claude Code Skill。

基于知网 3.0、万方、PaperPass 等主流检测器的技术原理，通过拆模板结构、打破均匀句式、嵌入操作细节、加入作者判断等方法，在保持学术语气的前提下让文本呈现「人类写作」的统计特征。

## 安装

```bash
# 克隆到 Claude Code skills 目录
git clone https://github.com/xiaofenggan01/aigc-reduce.git ~/.claude/skills/aigc-reduce
```

安装后，当你在 Claude Code 中提到「降 AI 味」「降 AIGC 率」「去 AI 检测」等关键词时，该 skill 会自动触发。

## 为什么需要这个工具？

AIGC 检测器已经在各大高校强制使用。但检测技术存在根本性局限：

- **写得好 = 判 AI？** 高质量流畅论文假阳性率 6.5%，有口语化瑕疵的论文仅 1.8%
- **跨平台结果不一致：** 同一篇论文知网 AI 率 35%，万方仅 12%
- **术语密集型论文受害：** 术语密度 >20% 时假阳性率 8-12%
- **检测器依赖浅层特征：** DPO 训练可使检测准确率下降 60%

本工具的核心思路：不是帮人作弊，而是帮**本来是人写的论文**降低被误判的风险。

## AIGC 检测是怎么工作的？

主流检测技术分为三大路线：

### 路线一：主动水印

在 AI 生成阶段嵌入隐蔽标识（如 KGW 算法、Google SynthID）。即使文本被改写，水印仍可溯源。

- 代表：Google SynthID（已整合进 Gemini）
- 弱点：修改比例 >40% 时假阴性率 60-80%

### 路线二：被动统计检测

AI 文本在统计特征上与人类文本有系统性差异。检测器主要抓以下指标：

| 指标 | AI 文本特征 | 人类文本特征 |
|------|-----------|------------|
| **困惑度（Perplexity）** | 低（选择概率最高的词，高可预测） | 高（个性化表达、创造性措辞） |
| **突发性（Burstiness）** | 低（句长均匀、句式规整） | 高（长短句交错、词汇波动大） |
| **模板化句式** | "基于…分析""综上所述""此外" | 自然叙述，较少固定搭配 |
| **段落对称性** | 相邻段落长度相近 | 段落长度波动明显 |

- 代表：GPTZero（困惑度 + 突发性）、知网语义指纹模块
- 弱点：对抗性改写可轻易规避；对术语密集型/规范化文本误判率高

### 路线三：深度学习判别

用 RoBERTa、DeBERTa 等预训练模型，在大规模标注数据上学习人类文本与机器文本的语义差异。

- 代表：知网 3.0（准确率 98.6%）、Turnitin AI Detection
- 弱点：跨模型泛化差；DPO 训练可使其准确率下降 60%；"黑箱"判别缺乏可解释性

### 知网 3.0 语义指纹模块

知网专门抓取以下 AI 文本模式：

1. **模板化句式** — "基于…分析""通过实验验证""综上所述"
2. **逻辑断层** — 段落间缺乏自然过渡
3. **数据模糊性表述** — "据统计""相关研究表明"，无具体来源
4. **标准化标题层级** — 过于规整的（1）（2）（3）编号
5. **格式规范性异常** — 参考文献格式错误

> 详细的检测原理分析见 [references/detection-principles.md](references/detection-principles.md)

## 四层降重方法

基于上述检测原理，本工具提供四个层级的改写方法，按效果由浅到深：

### 第一层：表层改写

适用于 AI 生成痕迹较轻的段落。

- 拆长句为短句（50 字以上拆为 2-3 句）
- 被动改主动（"被测定为" → "测得"）
- 同义词轮换
- 标点打破规律

**对应检测维度：** 突发性 CV、被动语态比例

### 第二层：结构重构

适用于检测报告中「高 AI 率」的段落。核心层。

- 拆模板化句式（嵌套编号 → 自然叙述、冒号并列 → 独立段落）
- 打破对称段长（连续 3 段以上长度相近时，故意扩写/精简）
- 加入段落间过渡（每段结尾加向下引导，或下段开头承接上文）

**对应检测维度：** 知网语义指纹、段落对称性、嵌套编号

### 第三层：语义注入

适用于核心检测段落。效果最强。

- 嵌入操作观察与波动（"过程中发现粉料容易结团""反应时间浮动 ±15 min"）
- 加入作者判断（"本研究结果显示""综合来看""这一现象值得注意"）
- 具体化模糊表述（删掉"据统计"，给出具体数值或来源）

**对应检测维度：** 困惑度、模板句式密度

### 第四层：专项针对

- **文献综述降 AI**（假阳性重灾区 9.2%）：避免"A 认为…B 指出…"的并列列举，改为有主次、有批判的叙评结合
- **结论与展望降 AI**：避免（1）（2）（3）（4）的对称列举，合并叙述

**对抗效果参考：**

| 修改程度 | 检测规避率 | 对应层级 |
|---------|-----------|---------|
| 轻度修改 (<20%) | 15-25% | 第一层 |
| 中度修改 (20-40%) | 35-50% | 第一层 + 第二层 |
| 深度修改 (>40%) | 60-80% | 四层全用 |

## 自动化扫描

内置 `aigc_scan.py` 脚本，自动扫描文本的 7 个 AI 特征维度：

```bash
python3 scripts/aigc_scan.py your_paper.txt
```

输出示例：

```
============================================================
  AIGC 特征扫描报告
============================================================

文本概况: 25000 字符, 120 段, 480 句

────────────────────────────────────────
  [1] 模板句式
      命中: 18 处 (密度: 0.72/千字)
      风险: ⚠️  偏高
  [2] 被动语态
      被动标记: 8 处 (0.017/句)
      风险: ✅ 正常
  [3] 句子长度变化（突发性）
      变异系数 CV: 0.28
      风险: ⚠️  高风险 — 句子长度过于均匀，典型 AI 特征
  ...
```

扫描维度与检测器的对应关系：

| 维度 | 对应检测器 |
|------|-----------|
| 模板句式密度 | 知网语义指纹（模板化句式） |
| 突发性 CV | GPTZero 核心算法（困惑度+突发性） |
| 段落对称性 | 知网格式规范性校验 |
| 嵌套编号 | 统计检测（结构模板化） |
| 冒号并列 | 知网逻辑断层检测 |
| 被动语态 | Turnitin 句式特征分析 |
| 标点规律 | 万方 BERT 语义分析 |

支持 `--json` 输出结构化数据：

```bash
python3 scripts/aigc_scan.py your_paper.txt --json
```

## 高误判场景（你的论文可能被误判）

| 文本类型 | 假阳性率 | 原因 |
|---------|---------|------|
| 专业术语密集型（术语 >20%） | 8-12% | 规范表达与 AI 特征相似 |

| 高质量流畅文本 | 6.5% | "完美输出"与 AI 生成高度重合 |
